*codex_bridge.nvim* Send Neovim context to a Codex app-server session

==============================================================================
COMMANDS                                              *codex_bridge-commands*

                                                        *:CodexStartSession*
:CodexStartSession
        Start a plugin-owned `codex app-server` on a Unix socket and open an
        terminal running `codex --remote unix://...`.

:CodexStartSession!
        Start the app-server without opening the external terminal.

                                                  *:CodexStartResumeSession*
:CodexStartResumeSession {session_id}
        Start the app-server, resume a known Codex session/thread id, and open
        the terminal with `codex resume --remote ...`.

                                                              *:CodexSend*
:[range]CodexSend [prompt]
        Send the selected range, or the current file when no range is given,
        together with the prompt. If [prompt] is omitted, `vim.ui.input` asks
        for one.

                                                       *:CodexStopSession*
:CodexStopSession
        Stop the plugin-owned app-server.

                                                            *:CodexStatus*
:CodexStatus
        Print current plugin state.

==============================================================================
LUA API                                                  *codex_bridge-lua*

Setup:
>
  require("codex_bridge").setup({
    terminal = {
      -- nil means auto-detect from $TERM, then fall back to common executables.
      command = nil,
    },
    keymaps = {
      enabled = true,
      prefix = "<leader>c",
    },
  })
<

Terminal command override:
>
  require("codex_bridge").setup({
    terminal = {
      command = { "your-terminal", "-e" },
    },
  })
<

Statusline:
>
  require("codex_bridge").statusline()
<

Full status:
>
  require("codex_bridge").status()
<

Subscribe to status changes:
>
  require("codex_bridge").on_status_change(function(state)
    -- state.status, state.busy, state.thread_id, state.socket_path
  end)
<

==============================================================================
KEYMAPS                                                *codex_bridge-keymaps*

Default keymaps are enabled by setup().

                                                        *codex_bridge-<leader>cs*
<leader>cs     Start session and open terminal.

                                                        *codex_bridge-<leader>cS*
<leader>cS     Start session without opening a terminal.

                                                        *codex_bridge-<leader>cr*
<leader>cr     Resume a session by thread id.

                                                        *codex_bridge-<leader>cc*
<leader>cc     Send current file in Normal mode, or selection in Visual mode.

                                                        *codex_bridge-<leader>cx*
<leader>cx     Stop session.

                                                        *codex_bridge-<leader>ci*
<leader>ci     Inspect status.

Disable defaults:
>
  require("codex_bridge").setup({
    keymaps = {
      enabled = false,
    },
  })
<
